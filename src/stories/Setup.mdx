# Styling

MinID elements provides a Tailwind theme based on [Designsystemet.](https://www.designsystemet.no/)

## Tailwind

See the [Tailwind documentation](https://tailwindcss.com/docs/installation) for information
on how to install Tailwind. We recommend PostCSS if you are using Custom Components/Lit Components.

To use the Tailwind theme in your application, simply import the theme in your `tailwind.css` file.

You can have a look at the raw theme file [here](https://github.com/felleslosninger/minid-web-components/blob/main/src/public/designsystemet-tailwind.css)

```css
// tailwind.css
@import 'tailwindcss';
@import '@felleslosninger/minid-elements/designsystemet-tailwind';
```

The tailwind file should be included in the head with `data-mid-tailwind`. This css sheet is inserted into the shadow dom of the custom comopnents.

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />

    <!-- Tailwind styles for the component library -->
    <link data-mid-tailwind href="src/styles/tailwind.css" rel="stylesheet" />
  </head>
  <body>
    <mid-button>Click me</mid-button>
  </body>
</html>
```

The consuming application should generate tailwind classes for minid-elements. The minid-elements dist folder should be included as a tailwind source so

```css
// tailwind.css
@import 'tailwindcss';
@import '@felleslosninger/minid-elements/designsystemet-tailwind';
@source '../../node_modules/@felleslosninger/minid-elements/dist';
```

MiniID elements also has some global CSS that should be imported.

```css
// index.css
@import '@felleslosninger/minid-elements/styles';

```

You can generate your own theme with designsystemet CLI.

More information on how to
install and set up Designsystemet themes can be found [here](https://github.com/digdir/designsystemet)
